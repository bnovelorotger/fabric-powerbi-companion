# Semantic warning rules

Usa este bloque cuando detectes ambiguedad o conflicto semantico:

```yaml
semantic_warning:
  object:
  warning:
  suggested_definition:
  requires_confirmation: true
```

## Generic warnings to watch

- Lead vs Applicant
- Applicant vs Admitted Customer
- Forecast vs Actual
- Revenue vs Cash Collected
- Enrolled User vs Active User
- Activity vs Outcome
- Retention vs Graduation Rate
